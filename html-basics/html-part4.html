Forms and Input

<form>

用途：創建供用戶輸入的HTML表單，可包含<input>、<textarea>、<button>、<select>、<option>、<optgroup>、<fieldset>、<label>
屬性：
  • action: 指定當提交表單時向何處發送表單數據
  • autocomplete: 指定是否啟用表單的自動完成功能
  • method: 指定用於發送表單數據的 HTTP 方法
  • target: 指定在何處打開 action URL

<input>

用途：讓使用者可以在其中輸入資訊
屬性：
  • accept: 指定通過檔案上傳來提交的檔案類型。(只針對type="file")
  • alt: 定義圖片輸入的替代文字。(只針對type="image")
  • checked: 指定在頁面載入時應該被預先選取的<input>元素。(只針對 type="checkbox" 或者 type="radio")
  • disabled: 指定禁用的<input>元素
  • height: 指定<input>元素的高度。 (只針對type="image")
  • max: 指定<input>元素的最大值
  • min: 指定<input>元素的最小值
  • maxlength: 指定<input>元素中允許的最大字元數
  • name: 指定<input>元素的名稱
  • placeholder: 指定可描述輸入<input>字段預期值的簡短的提示
  • readonly: 指定輸入字段是唯讀的
  • required: 指定必需在提交表單之前填寫輸入字段
  • size: 以字元數計的<input>元素的可見寬度
  • src: 指定顯示為提交按鈕的圖片的URL。(只針對 type="image")
  • value: 指定<input>元素 value 的值
  • width: 指定<input>元素的寬度。(只針對type="image")
  • type: 指定要顯示的<input>元元素的類型，包括
    • button:
    • checkbox:
    • color:
    • date:
    • datetime:
    • datetime-local:
    • email:
    • file:
    • image:
    • month:
    • number:
    • password:
    • radio:
    • range:
    • reset:
    • search:
    • submit:
    • tel:
    • text:
    • time:
    • url:
    • week:
範例：一系列的單選選項(同時只有一個能被選取)

<textarea>

用途：定義一個多行的文字輸入區
屬性：
  • cols: 指定文字區域內可見的寬度
  • rows: 指定文字區域內可見的行數
  • disabled: 指定禁用文字區域
  • form: 定義文字區域所屬的一個或多個表單
  • maxlength: 指定文字區域允許的最大字元數
  • name: 文字區域的名稱
  • placeholder: 一個簡短的提示，描述文字區域期望的輸入值
  • readonly: 指定文字區域為唯讀
  • required: 指定文字區域為必填的
  • wrap: 指定當提交表單時，文字區域中的文字應該怎樣換行，其值可以是hard或soft
範例：顯示一個文字輸入區及一個提交按鈕

<button>

用途：定義一個按鈕。與使用<input>創建的按鈕不同處在於，在<button>內部可以放置文字或圖片等內容
屬性：
  • disabled: 禁用此按鈕
  • form: 定義按鈕所屬的一個或多個表單
  • name: 按鈕的名稱
  • type: 按鈕的類型，其值可以是button、reset或submit
  • value: 指定按鈕的初始值。可由腳本進行修改
範例：定義一個按鈕

<select>

用途：創建下拉式選單，與<option>一起使用
屬性：
  • disabled: 禁用此選單
  • form: 定義此選單所屬的一個或多個表單
  • multiple: 指定可以選擇多個選項
  • name: 選單的名稱
  • required: 規定使用者在提交表單前必須選擇一個下拉列表中的選項
  • size: 指定下拉列表中可見選項的數目

<option>

用途：定義下拉式選單中的一個選項（一個條目）
屬性：
  • disabled: 定義此選項應在首次載入時被禁用
  • selected: 指定選項（在首次顯示在列表中時）為選中狀態
  • value: 定義選項的值
範例：定義兩個單選的下拉式選單和一個多選的下拉式選單

<label>

用途：為input元素定義標註
屬性：
  • for: 指定label與哪個表單的元素綁定
  • form: 指定label所屬的一個或多個表單

<fieldset>

用途：將表單內的相關元素分組，在元素周圍繪製邊框
屬性：
  • disabled: 指定該區塊中的相關表單元素應該被禁用
  • form: 指定區塊所屬的一個或多個表單。
  • name: 指定區塊的名稱

<legend>

用途：為<fieldset>元素定義標題

Styles and Semantics

<style>

用途：定義HTML檔案的樣式信息
屬性：
  • media: 為stylesheet指定不同的媒體類型
  • scoped: 指定樣式僅僅應用到 style 元素的父元素及其子元素
  • type: 規定stylesheet的MIME類型
註記：如果沒有使用"scoped"屬性，則每一個<style> tag必須位於<head>

<div>

用途：定義 HTML 文檔中的一個分隔區塊或者一個區域部分，常用於組合Block Level Element

<span>

用途：用於對文檔中的Inline Element進行組合

<section>

用途：定義了文檔的某個區域。比如章節、頂部、底部或者文檔的其他區域

<article>

用途：定義獨立的內容，內容本身必須是有意義的且必須是獨立於文檔的其餘部分，例如文章、新聞內文

<aside>

用途：定義<article>標籤外的內容，aside的內容應該與附近的內容相關，通常用作文章的側欄

Meta Info

<meta>

用途：<meta>提供了HTML文檔的元數據。元數據不會顯示在客戶端，但是會被瀏覽器解析，該tag通常用於指定網頁的描述，關鍵詞，文件的最後修改時間，作者及其他元數據
屬性：
  • charset: 定義文檔的character encoding
  • content: 定義與http-equiv或name屬性相關的元數據，其值可以是content-type、default-style、或refresh
  • name: 把 content 屬性關聯到一個名稱，其值可以是application-name、author、description、generator、或keywords
註記：<meta>標籤通常位於<head>區域內
註記：元數據通常以鍵值對出現
註記：元數據可以被瀏覽器（如何顯示內容或重新加載頁面），搜索引擎（關鍵詞），或其他 Web 服務調用

<base>

用途：為頁面上的所有的相對連結指定預設URL或預設目標
屬性：
  • href: 指定頁面中所有相對連結的基準URL
  • target: 指定頁面中所有的連結和表單在何處打開。該屬性會被每個連結中的target屬性覆蓋
註記：在一個文檔中，最多能使用一個<base>元素。<base>必須位於<head>內部
